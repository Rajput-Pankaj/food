# FoodExpress ‚Äî Food Delivery Platform

Full-stack food delivery application with a **React 19** frontend, **Express API**, and **PostgreSQL** database. Deploy in one command with Docker, or run locally with browser storage for quick demos.

## Features

- **Marketing homepage** ‚Äî hero slider, offers, testimonials, featured dishes
- **50-item menu** ‚Äî nutrition, allergens, galleries, dietary filters
- **Blog** ‚Äî 8 seeded posts, admin CRUD
- **Customer dashboard** ‚Äî orders, profile, addresses, reviews
- **Checkout** ‚Äî delivery / takeaway, COD, dynamic UPI QR, Razorpay
- **Order management** ‚Äî accept / reject, full status timeline, live tracking
- **Admin panel** ‚Äî dashboard, orders, menu, users, blogs, reviews, store settings

## Quick Start ‚Äî One-Command Deploy (Recommended)

Clone the repo on any VPS with Docker and run:

```bash
curl -fsSL https://raw.githubusercontent.com/pankajkumarrajput1116-glitch/food/main/scripts/install.sh | bash
```

Or manually:

```bash
git clone https://github.com/pankajkumarrajput1116-glitch/food.git foodexpress
cd foodexpress
./scripts/deploy.sh
```

Then open **`/setup`** in your browser ‚Äî **no setup token to paste**. The wizard auto-verifies the server, configures the store, admin account, and optional domain (Traefik HTTPS).

The deploy script automatically:

- Checks system requirements (Docker, memory, disk)
- Detects **Traefik** for automatic HTTPS (when `DOMAIN` is set)
- Generates `.env` with strong random secrets and `SETUP_TOKEN`
- Builds and starts PostgreSQL + API + Nginx web
- Runs DB migrations on API startup

**First visit:** open `/setup` ‚Äî automated wizard (no token paste). Optional domain step writes `.env` and prompts `./scripts/deploy.sh` for Traefik HTTPS.

**Other deploy options:**

| Method | Command / action |
|--------|------------------|
| One-line install | `curl -fsSL ‚Ä¶/scripts/install.sh \| bash` |
| SSH one-command | `./scripts/deploy.sh` |
| Docker Compose only | `docker compose up --build -d` |
| Coolify / Hostinger Git | Paste repo URL + set env vars (see [DEPLOYMENT.md](docs/DEPLOYMENT.md)) |
| Generate secrets | `./scripts/generate-secrets.sh` |
| Remove stack | `./scripts/undeploy.sh` or `./scripts/undeploy.sh --purge` |

**With Traefik SSL:**

```bash
DOMAIN=food.yourdomain.com ./scripts/deploy.sh
```

**Manual Docker (legacy):**

```bash
cp .env.example .env
npm run docker:up
open http://localhost:8080
```

**What runs:**

| Service | Description | Port |
|---------|-------------|------|
| `web` | Nginx ‚Äî React SPA + `/api` proxy | 8080 |
| `api` | Express REST API | internal 3001 |
| `db` | PostgreSQL 16 | internal 5432 |

**Useful commands:**

```bash
npm run docker:logs    # Follow container logs
npm run docker:down    # Stop and remove containers
```

On first boot the API runs migrations and seeds menu (50 items), blogs, and store settings. Demo admin/customer accounts are created only when `SEED_DEMO_USERS=true` (default `false` in production).

## Local Development

### Option A ‚Äî Frontend only (no database)

Uses **localStorage** for all data. Great for UI work.

```bash
npm install
npm run dev
```

Open http://localhost:5180 ‚Äî no backend required.

### Option B ‚Äî Frontend + API + PostgreSQL

```bash
# Terminal 1 ‚Äî database + API (hot reload)
npm run docker:dev

# Terminal 2 ‚Äî frontend pointing at API
echo "VITE_API_URL=http://localhost:3001/api" >> .env.local
npm run dev
```

Or run the API directly on your machine:

```bash
# Start Postgres (Docker)
docker compose -f docker-compose.dev.yml up db -d

# Install API deps and start
cd server && npm install && npm run dev

# Frontend
VITE_API_URL=http://localhost:3001/api npm run dev
```

## Environment Variables

Copy `.env.example` to `.env`:

| Variable | Description | Default |
|----------|-------------|---------|
| `APP_PORT` | Public web port | `8080` |
| `POSTGRES_USER` | Database user | `foodexpress` |
| `POSTGRES_PASSWORD` | Database password | *(change me)* |
| `POSTGRES_DB` | Database name | `foodexpress` |
| `JWT_SECRET` | API auth secret | *auto-generated by deploy* |
| `JWT_EXPIRES_IN` | Access token lifetime | `15m` (see `.env.example`; deploy script may use `7d`) |
| `CORS_ORIGIN` | Allowed origin | auto from `DOMAIN` or localhost |
| `DOMAIN` | Public domain (Traefik SSL) | optional |
| `SEED_DEMO_USERS` | Create demo admin/customer | `false` in production |
| `SETUP_TOKEN` | First-time setup token | auto-generated by deploy |
| `ALLOW_REGISTRATION` | Public signup | `true` (set `false` to disable) |
| `SMTP_HOST` | Email notifications | optional |
| `APP_URL` | Public app URL for reset links | falls back to `CORS_ORIGIN` |
| `RAZORPAY_KEY_SECRET` | Razorpay payment verify | optional |
| `VITE_API_URL` | Frontend API base | empty = localStorage mode |

See [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md) for VPS, cloud, and manual setup.

## API Overview

When `VITE_API_URL` is set, the frontend talks to:

| Endpoint | Purpose |
|----------|---------|
| `POST /api/auth/login` | Login |
| `POST /api/auth/register` | Signup |
| `GET /api/orders` | List orders |
| `POST /api/orders` | Place order |
| `GET /api/menu` | Public menu |
| `GET /api/settings` | Store settings |
| `GET /api/blogs` | Published posts |
| `GET /api/health` | Health check |

Admin routes require a JWT with the `admin` role.

## Order Status Flow

```
pending ‚Üí accepted ‚Üí preparing ‚Üí ready ‚Üí shipped ‚Üí delivered
```

Takeaway orders skip `shipped`. Admins can **reject** pending orders.

- Razorpay signature / payment verification
- Rate limiting + Helmet security headers
- Strong secrets required in production (auto-generated by deploy script)
